# Daily Briefing

## When to Use
Run this skill each morning (or when the user asks for a briefing). If on a schedule, trigger at the start of the workday.

## Process

1. **Check today's calendar.** List all meetings and events with times, attendees, and purpose.

2. **Surface top priorities.** Pull from memory and task lists:
   - What are the 3 most important things to accomplish today?
   - Any deadlines hitting today or tomorrow?
   - Any follow-ups from yesterday that are still open?

3. **Flag items needing attention:**
   - Meetings with no agenda or prep
   - Conflicts or back-to-back blocks with no buffer
   - Overdue tasks or approaching deadlines
   - Pending decisions or approvals

4. **Provide a weather check on the week** (brief):
   - Any major events or deadlines later this week worth keeping in mind today?

5. **End with a clear recommendation:**
   - "I'd suggest starting with [X] before your 10am meeting."
   - Keep it to one actionable suggestion.

## Format
```
Good morning. Here's your briefing for [day, date].

Schedule:
- 9:00  — [Event] with [people]
- 10:30 — [Event] with [people]
- ...

Top priorities:
1. [Most important task]
2. [Second priority]
3. [Third priority]

Needs attention:
- [Flag: e.g., "No agenda set for 2pm board sync"]
- [Flag: e.g., "Proposal draft due tomorrow — not started"]

Suggestion: Start with [X] — it's the highest-leverage item before your first meeting.
```

## Tone
Crisp and informative. No filler. The user should be able to scan this in 30 seconds and know exactly what their day looks like.
