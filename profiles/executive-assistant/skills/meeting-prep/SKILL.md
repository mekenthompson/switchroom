# Meeting Prep

## When to Use
Run this skill before important meetings — ideally 30-60 minutes before, or when the user asks to prep for a specific meeting.

## Process

1. **Identify the meeting.** Confirm which meeting you're prepping for: name, time, attendees.

2. **Gather context:**
   - Who's attending? Pull any stored context about these people from memory (roles, last interaction, relationship notes).
   - What's the agenda? If one exists, summarize it. If not, suggest one based on context.
   - What happened last time? Check memory for notes from previous meetings with the same group.
   - Any open items or commitments the user made that are relevant?

3. **Compile the briefing:**
   - **Purpose**: One sentence on why this meeting is happening.
   - **Attendees**: Names, roles, and any relevant context (e.g., "Sarah — pushed back on timeline last meeting").
   - **Agenda**: Existing or suggested.
   - **Key context**: Relevant background the user should have top of mind.
   - **Open items**: Anything the user committed to that they should update on.
   - **Suggested talking points**: 2-3 things the user might want to raise.

4. **Ask if the user wants to add or adjust anything** before the meeting.

5. **After the meeting** (if the user reports back): capture action items, decisions, and any follow-ups in memory.

## Format
```
Prep: [Meeting name] — [time]

Purpose: [One sentence]

Attendees:
- [Name] — [Role/context]
- [Name] — [Role/context]

Agenda:
1. [Item]
2. [Item]
3. [Item]

Context:
- [Relevant background point]
- [Previous decision or open item]

Suggested talking points:
- [Point the user might want to raise]
- [Question to ask]
```

## Tone
Briefing-style. Think of it as a one-page prep document a chief of staff would hand over before a meeting. Factual, organized, actionable.
